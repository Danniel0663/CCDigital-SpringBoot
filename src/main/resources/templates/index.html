Cómo funciona

Flujo pensado para que el usuario entienda y controle qué entidad accede a su información.

1

Registro y validación del correo

El usuario completa sus datos, verifica el correo con un código y puede activar autenticador móvil desde el inicio.

2

Ingreso seguro

Se validan credenciales, credencial digital y un segundo factor (app autenticadora o código por correo).

3

Consentimiento explícito

La entidad emisora solicita documentos y el ciudadano decide aprobar o rechazar con fecha y trazabilidad registrada.

4

Trazabilidad y consulta

Los documentos aprobados se validan y sincronizan para garantizar integridad, origen y control de visualización.

Seguridad y trazabilidad del proyecto

La plataforma combina controles de aplicación con verificación de identidad y registro de evidencia.

Contraseñas protegidas

Las contraseñas se almacenan con hash BCrypt y nunca se guardan en texto plano.

Credenciales verificables

El ingreso de usuario valida credenciales digitales antes de completar la autenticación de sesión.

MFA por correo y app

Se soporta doble factor con códigos temporales por correo y TOTP compatible con apps de autenticación.

Trazabilidad on-chain

La integración con Hyperledger Fabric permite validar la existencia y origen de documentos aprobados.
